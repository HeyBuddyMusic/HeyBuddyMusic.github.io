WORK ABOUT MUSIC

DrumScript

A tiny javascript drum machine.

As a student in General Assembly's web development immersive, one of our first projects was to create a game using CSS, HTML and DOM manipulation via "vanilla" JavaScript. My project was a rhythm dictation game- something of a variation on "Simon".

Challenges

The challenge we faced was finding the right criteria to use to exclude restaurants by category. We decided that the greatest challenge was to let the user say, something to the effect of, "I don't want italian food tonight." We also had to account for a user who rejected all possible options around them.

Solution

Diving into Yelp's "Fusion API" revealed that each restaurant was assigned tags based on the type of food and drink they served (e.g. "Bars, American (Traditional)"). We decided to have the app present the user with the option to either say that they didn't like the individual restaurant or they didn't like the type of food presented to them. To implement this, we had Yelp retrieve the 50 closest restaurants to the user's position and stash them in local storgae. Each time one was selected at random, it would check to see if either the title or any categories were selected and move on to another if either criteria were met. We also set the app up to admonish the user for being whiny if they had exhausted all possible options.

Did It Work?

Sort of. The app functions great with a large sample size of restaurants- the sort of thing that a major city like NYC is great for. I handed it my hometown in rural Maine out of curiosity and ended up being scolded by the app after turning down all five options handed to me.

What would you do differently?

Given more time, it would be nice to include other exclusionary features. These might be things like giving the user an option of specifying how far they're willing to travel. Otherwise, I'm not only happy with it, I actually use this pretty consistently, which is the nicest thing that I can say about anything I've built!
